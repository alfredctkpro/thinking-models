# AI 研究工具的選擇與搭配

## 1. AI 研究工具

### Perplexity — 事實面研究的首選

進行偏向「事實面」的 AI 研究時，Perplexity 是目前最適合的工具。它能快速提供有來源引用的回答，適合查證、比較與釐清事實。

Perplexity 內建的 Computer 功能（自動化操作）目前不太實用，不夠實際。

### Manus / Genspark — 需要搭配事實查核

如果要使用 Manus 或 Genspark 這類工具來做研究，可能需要搭配「事實查核」的 Agent 才能確保資訊品質。但目前不確定 Manus 和 Genspark 是否有這樣的 Agent 工具可以運用。

## 2. NotebookLM — 鎖定研究範圍與邊界

當有特定題材要研究時，可以用 NotebookLM 把資料整理進去，運用它來進行學習。NotebookLM 的優勢在於能夠鎖定研究的範圍與邊界，避免發散。

## 3. 工具選擇的思考原則

### YouMind 等工具：目前不需要

也在思考是否需要像 YouMind 這樣的工具來進行題材研究，但目前的結論是不需要。原因：

1. 我在吸收、學習或研究新東西時，並不是透過廣泛搜尋網頁來記錄重點。
2. 就算要記錄，也不是透過「劃線」的方式。
3. 再增加新工具，對思考、學習與進化的過程來說，反而增加了太多的**外部認知負荷**——這不是好主意。

### 核心原則：降低外部認知負擔

我們希望降低外部認知的負擔，包括工具的使用部分。這與[主動高認知負荷](../models/成長戰略/主動高認知負荷.md)的觀點一致——要把認知資源投入在「增益負荷」（真正讓你成長的部分），而非「外在負荷」（工具切換、格式轉換帶來的消耗）。

## 目前的結論

**現有搭配已經足夠：**

```
Perplexity（事實查證）→ NotebookLM（特定題材深入學習）
```

## 待解決：筆記與資料回找

目前缺乏的環節是「筆記相關」或「後續資料回找」的部分。

- 能更緊密地與整個思考體系結合

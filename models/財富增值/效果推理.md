# 效果推理——從手中之鳥到駕駛艙裡的飛行員

## 一句話摘要

當你不知道要做什麼的時候，從手頭現有的資源出發，做出能改變下一步的小結果；效果推理是一種反脆弱戰略，把擁有物變成資源、把關係鏈變成承諾、把意外變成方向。

## 模型說明

「效果推理（Effectuation）」由印度裔美籍管理學者 **Saras Sarasvathy** 提出。她研究了大量連續創業成功者，發現他們的思考方式與一般 MBA 教的「規劃式創業」截然不同——他們用的是「效果推理」。

### 因果推理 vs 效果推理

| 維度 | 因果推理（Causal）| 效果推理（Effectual）|
|------|------|------|
| 起點 | 已知目標 | 已知手段 |
| 方向 | 目標 → 手段 | 手段 → 目標 |
| 比喻 | 賈伯斯做 iPhone：已經知道要什麼、再去設計 | 打開冰箱看到雞蛋、火腿、米飯 → 做蛋炒飯 |
| 對待意外 | 視為干擾，要排除 | 視為機會，要擁抱 |
| 對待夥伴 | 分析對手 | 尋找盟友 |
| 評估標準 | 預期回報 | 可承受損失 |
| 控制觀 | 預測未來 | 創造未來 |

#### 因果推理的例子

- 想做「宮保雞丁」→ 去買雞肉、花生、辣椒 → 按食譜做
- 賈伯斯推 iPhone：已知道要什麼產品 → 整合所需技術與供應鏈
- 一般 MBA 教法：先寫商業計畫書 → 找市場 → 配置資源 → 執行

#### 效果推理的例子

- 打開冰箱看到雞蛋、火腿、米飯 → 決定做蛋炒飯
- 連續創業家：先盤點手上有什麼 → 邊做邊找方向 → 持續調整
- 早期創業：市場根本還不存在，沒辦法計算回報，只能從手邊資源開始

> 在**目標已知**的世界裡用因果推理；在**目標未知**的世界裡用效果推理。

### 效果推理的五個原則

#### 1. 手中之鳥（Bird in Hand）

> 「一鳥在手勝過十鳥在林。」

從你**現有的資源**出發，再慢慢推導出可能的目標。資源分三層：

- **你是誰（Who you are）**：性格、偏好、經驗、價值觀
- **你會什麼（What you know）**：專業技能、工作經驗、領域知識
- **你認識誰（Who you know）**：朋友圈、社群關係——誰願意為你投入時間、金錢、見識、信譽

這三層資源組合起來，就是你的**起點地圖**。不要先想「我應該做什麼」，要先盤點「我手上有什麼」。

#### 2. 可承受的損失（Affordable Loss）

因果推理問：「這個項目能賺多少？」
效果推理問：「**如果失敗，我能不能承受？**」

為什麼？因為在新市場、新領域，**你根本沒辦法準確預估回報**。所有「五年後年收 X 億」的試算都是幻覺。

但你可以準確估計：**最壞情況下，我會損失多少？**

只要這個損失在你能承受的範圍內，就可以開始；損失若超過你能承受，再美好的回報也不該下注。

> 這呼應凱利公式的精神：不能 all-in。

#### 3. 瘋狂拼布（Crazy Quilt）

因果推理會**分析競爭對手**；效果推理會**尋找合作夥伴**。

具體做法：

- 儘管還不知道最終要做什麼，**儘快開始跟可能的夥伴談**
- 看誰願意把資源（時間、錢、能力、人脈）一起拼進來
- 每一個 **Commitment（承諾）** 都是一塊新拼布

承諾越多，你的拼布越完整——即使市場還沒找到，這些承諾本身就會大幅提高成功機率。

> 創業不是一個人的英雄主義，而是不斷把更多人拉進這張拼布。

#### 4. 檸檬水原則（Lemonade）

源自美國諺語：「**If life gives you lemons, make lemonade.**」

- 因果推理**怕意外**——意外會打亂計畫
- 效果推理**擁抱意外**——意外可能是新方向、新入場券

實際情況中，很多成功的創業故事是：
- 原本要做 A，做著做著發現使用者最愛的是某個小功能 → 轉型做這個小功能
- 客戶要的不是你以為的東西 → 換產品方向
- 意外的合作機會找上門 → 進入新賽道

> 計畫之外發生的事，往往才是真正的線索。

#### 5. 駕駛艙裡的飛行員（Pilot in the Plane）

- 因果推理：**預測未來**——假設世界有固定航線，越精準預測越好
- 效果推理：**創造未來**——假設世界是動態的，靠**臨場控制**

像飛機駕駛員：
- 你**不能控制**天氣、空中亂流、其他飛機
- 但你**能控制**燃油、方向、速度、起降時機

**你的行動才是決定航向的關鍵變量**——不是預測，是控制。

> 與其爭辯「未來會怎樣」，不如專注「我下一步要做什麼」。

### 創業版的 OODA 循環

五個原則可以對應到 OODA 循環：

```
觀察（Observe）   ← 手中之鳥（盤點現有資源）
定向（Orient）    ← 檸檬水原則（解釋意外、重新定位）
決定（Decide）    ← 可承受的損失（決定投入規模）
行動（Act）       ← 瘋狂拼布（拉夥伴、發動承諾）
       ↓
   再回到 Observe（手中之鳥又更新了）
```

> 效果推理 = 在不確定中跑 OODA 循環 + 透過行動本身去獲取新訊息。

### 效果推理的本質：反脆弱戰略

效果推理的底層精神是**反脆弱**：

- **有限下行**（可承受的損失）
- **無限上行**（檸檬水原則接住意外的機會）
- **越動越強**（每次行動都增加你的拼布塊）
- **與世界互動產生訊息**（不是預測世界，而是探測世界）

> 從擁有物變成資源、從關係鏈變成承諾、從意外變成方向——這就是反脆弱在創業上的具體做法。

### 什麼時候用因果、什麼時候用效果

不是「效果推理永遠比較好」——兩種思維各有適用場景：

| 情境 | 適合的推理 |
|------|-----------|
| 市場已成熟、目標明確、規則清楚 | **因果推理**（規劃、執行、優化）|
| 市場未形成、目標不明、規則模糊 | **效果推理**（手中之鳥、檸檬水、拼布）|
| 大型企業導入新流程 | **因果推理** |
| 個人創業初期、副業探索 | **效果推理** |
| AI 時代的新賽道進入 | **效果推理**（沒人知道未來會怎樣）|

對應[機會窗口](機會窗口.md)的五階段：
- **暗流期 / 風起於青萍之末**：必須用效果推理（市場還沒形成）
- **最佳入場窗口**：兩者並用（有方向但變數還很多）
- **如日中天 / 天已過午**：偏因果推理（規則已定，靠執行效率）

## 使用場景

- **想創業卻不知道做什麼**：不要再寫商業計畫書，先盤點「你是誰、你會什麼、你認識誰」
- **副業探索**：不問「能賺多少」，問「失敗了我能承受多少」，從小規模開始
- **轉職迷惘期**：手中之鳥盤點現有資源，然後用「瘋狂拼布」開始跟人聊
- **遇到意外/挫折**：問自己「這個意外有沒有可能是檸檬水的原料？」
- **在大公司內部創新**：手中之鳥（這個團隊有什麼）、可承受損失（不要 all-in 公司資源）、瘋狂拼布（拉跨部門夥伴）
- **教育/育兒**：孩子未來方向不明時，幫他盤點手中之鳥，鼓勵他用可承受損失做小實驗
- **AI 時代的新賽道**：沒人知道 AI 應用的標準答案，所以幾乎所有 AI 創業都該用效果推理

# 表徵・圖式・心智模型・解釋框架——理解世界的四個認知層次

## 一句話摘要

學習的本質是把世界壓縮成一套能夠預測、解釋、幹預、自我糾錯的內在地圖；這張地圖由表徵構成、由圖式組織、由心智模型驅動、由解釋框架統攝。

## 模型說明

理解一件事物，並不是「把資訊存入大腦」這麼簡單。我們的認知是有層次的，從最基礎的感知符號到最高層的領域全圖，分為四個遞進的層次：

### 層次一：Representation（表徵）

**事物在大腦中的代用品。**

世界是原子構成的，但大腦不處理原子。我們感知到的每一樣東西，都是一個「表徵」——一個能代表這個東西的內在符號。

- 蘋果不是原子堆，它在你腦海裡是「紅色圓圓的、甜甜的、可以吃的」這組感知集合
- 「AI Agent」在你腦海裡是一組文字、聲音、畫面、情感記憶的組合
- 表徵可以是視覺的、語言的、身體感受的、情緒的

表徵是認知的**原材料**。你的表徵越豐富、越精準，你就越能感知到更多細節——這正是天賦的一部分（參見刻意練習）。

### 層次二：Schema（圖式）

**把多個 Representation 組合在一起的快速識別模板。**

當你能從一組表徵中快速辨認出「這是一個整體」，你就在使用 Schema。

- 四隻腳 + 一片木板 → 你知道這是一張桌子，不需要逐一分析每個零件
- 電腦主機 + 螢幕 + 鍵盤 + 滑鼠 + 升降桌 → 一眼看出這是一個工作站
- 看到特定的句式、語氣、行文結構 → 辨識出這是哪種類型的文章

Schema 是**認知壓縮的基本單位**——它讓你不用每次從零開始理解，而是調用已有的模板。這也是認知負荷理論的核心：擁有 Schema 的人，處理新資訊時工作記憶的消耗大幅降低。

### 層次三：Mental Model（心智模型）

**解釋事物運作機制的動態結構。**

Schema 讓你認出「這是什麼」，Mental Model 讓你理解「這是怎麼運作的」。

- 知道 AI Agent 是什麼（Schema），和理解它背後怎麼運作（Mental Model）是兩件事
- 知道什麼是 Harness Engineering，和理解整套 Harness 的運作機制是兩件事
- 知道心臟是身體的器官，和理解血液循環的完整機制是兩件事

擁有正確的 Mental Model，你能**預測**事情會怎麼發展、**解釋**已發生的現象、**介入**並改變結果。

> Mental Model 是「跑得起來的機制」——它不只是描述，而是能生成推論的動態結構。

### 層次四：Explanatory Framework（解釋框架）

**對整個領域的全局理解，包含多個 Mental Models 與 Schemas 的完整系統。**

這是最高的認知層次。一個 Explanatory Framework 讓你能對整個知識領域做全局式的掌握。

- 不只知道幾個資安的 Schema 和 Mental Model，而是理解整個資安領域的結構——哪些地方是核心、哪些是邊界、各部分怎麼相互影響
- 不只知道幾個行銷技巧，而是理解整個行銷的底層邏輯——為什麼某些策略有效、什麼時候會失效
- 不只知道一些程式語言語法，而是理解整個系統架構——如何設計、如何取捨

Explanatory Framework 讓你能**舉一反三**——遇到任何新的情境，都能從整體理解中找到對應的切入點。

### 學習 = 建立內在地圖

這四個層次共同構成你的**認知地圖**：

```
Representation → Schema → Mental Model → Explanatory Framework
（感知符號）    （識別模板） （運作機制）    （全域系統圖）
```

**好的內在地圖具備四種能力：**

1. **預測**：能推斷尚未發生的事
2. **解釋**：能理解已發生的事
3. **幹預**：能改變事物的走向
4. **自我糾錯**：能在預測失敗後更新地圖

沒有這四種能力的「知識」，只是資訊的堆積，不是真正的理解。

### 兩種學習方向

這個框架支持兩種互補的學習路徑：

#### 由下而上（Bottom-Up）

從 Representation 開始，逐步建立 Schema，再理解 Mental Model，最後形成 Explanatory Framework。

- 適合：**初學者進入新領域**
- 方式：先認識名詞概念（表徵），再理解常見組合（圖式），再搞懂運作機制（心智模型），最後整合全局（解釋框架）
- 類比：「把書從薄讀到厚」——逐步積累細節，最終看到全貌
- 例：準備資安考試，先從名詞開始，接著理解各個 Schema（攻擊手法的組合），再理解各類攻擊的 Mental Model（為什麼有效），最後建立整個資安領域的 Explanatory Framework

#### 由上而下（Top-Down）

先建立 Explanatory Framework 的大致輪廓，再逐步填入 Mental Models 與 Schemas 的細節。

- 適合：**有基礎知識要系統化整合**
- 方式：先掌握整個領域的結構圖，再往下探索細節
- 類比：「把書從厚讀到薄」——先壓縮出全局，再用舉一反三應對各種場景
- 例：把大量資料餵給 AI（如 NotebookLM），請它梳理這個領域裡包含哪些 Mental Models、它們之間的關係、哪些地方還有疑問——由全局出發，再往下深挖

> **最理想的學習**是能在兩個方向靈活切換：先由下而上積累，再由上而下整合，最後能在任意層次之間自由移動。

### 「把書從厚讀到薄，再從薄讀到厚」

- **從厚到薄（壓縮）**：把整個知識體系提煉成 Explanatory Framework——知道這個領域的核心 Mental Models 是什麼、怎麼組織的
- **從薄到厚（展開）**：遇到任何具體場景，能從 Explanatory Framework 往下展開，找到對應的 Mental Model 和 Schema，從容應對

這個過程需要你的認知地圖是**可操作的**，而不只是「記住了很多東西」。

## 使用場景

- **評估自己真正理解了什麼層次**：問「我對這件事有 Schema？Mental Model？還是 Explanatory Framework？」
- **找到學習盲點**：如果你只能「認出」但無法「解釋運作」，你停在 Schema 層，需要往上建立 Mental Model
- **設計學習路徑**：決定從 Bottom-Up 還是 Top-Down 開始，或用 AI 工具加速 Explanatory Framework 的建立
- **與 AI 協作學習**：把大量資料餵給 AI，請它幫你梳理整個領域的 Mental Models 和 Framework，再自己往下深挖細節
- **教學設計**：先讓學生建立正確的 Representation 和 Schema，再帶他們理解 Mental Model，最後幫助他們建立 Explanatory Framework

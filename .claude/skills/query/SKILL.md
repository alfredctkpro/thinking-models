# Query — 知識庫查詢

搜尋 models/ 和 journal/ 中所有相關內容，綜合回答使用者的問題，並引用來源。

## 執行步驟

### 1. 解析查詢意圖

判斷使用者想要的是：
- **主題查詢**：「我之前學過什麼跟決策相關的？」→ 廣泛搜尋
- **特定模型**：「WOOP 是什麼？」→ 定位特定檔案
- **跨模型比較**：「安全感和身份認同有什麼關聯？」→ 讀取多個檔案並分析
- **日記回溯**：「我上週有什麼想法？」→ 搜尋 journal/

### 2. 搜尋知識庫

使用 Grep 和 Glob 搜尋相關內容：

```
搜尋範圍：
- models/**/*.md（所有思維模型）
- journal/**/*.md（所有日記）
- inbox/**/*.md（未整理的草稿）
- whiteboard/**/*.md（白板筆記）
```

搜尋策略：
1. 先用關鍵字 Grep 搜尋標題和內容
2. 找到相關檔案後，Read 完整內容
3. 追蹤「相關模型」連結，擴大搜尋範圍

### 3. 綜合回答

以結構化的方式回答使用者的問題：

```markdown
## 查詢結果：[主題]

### 直接相關的模型
- **[模型名稱](路徑)** — 摘要 + 與查詢的關聯

### 日記中的相關提及
- **[日期](路徑)** — 相關段落摘要

### 綜合分析
（如果查詢涉及比較或分析，在此提供跨模型的洞察）
```

### 4. 可選：寫回 wiki

如果查詢產生了有價值的綜合洞察，詢問使用者是否要：
- 建立一個新的彙整頁面（例如 `models/主題比較.md`）
- 更新現有模型的「個人反思」區段
- 記錄到 log.md

## 報告格式

回答時務必：
- 引用具體的檔案路徑，讓使用者能直接點擊查看
- 區分「模型原文說的」和「我的分析」
- 如果知識庫中沒有相關內容，坦白告知

## 注意事項

- 所有回答使用繁體中文（台灣）
- 不要從知識庫外部補充資訊，除非使用者明確要求
- 查詢結果基於知識庫現有內容，不捏造不存在的模型或日記

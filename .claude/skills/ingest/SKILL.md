# Ingest — 知識整理

將使用者提供的筆記、草稿或 raw material 整理成結構化的思維模型檔案，並維護索引和交叉連結。

## 輸入來源（任一皆可）

1. **口述/對話**：使用者直接描述學到的內容
2. **inbox/ 草稿**：使用者先在 inbox/ 寫了粗略筆記
3. **Raw material**：文章、逐字稿、PDF 等外部素材（未來）

## 執行步驟

### 1. 理解內容

- 讀取使用者提供的內容或指定的 inbox/ 檔案
- 如果是 raw material，先提取核心概念和要點
- 確認這是一個新模型、還是對現有模型的補充

### 2. 判斷分類

掃描 `models/` 的分類資料夾，判斷內容屬於哪個分類：
- 基本世界觀、成長戰略、決策判斷、學習教育、財富增值
- 參與者、領導者、演化者、高觀點

如果不確定，詢問使用者。

### 3. 建立或更新模型檔

**新模型：** 讀取 `models/_template.md`，在對應分類資料夾建立新檔案：

```markdown
---
title: 模型名稱
source: 萬維鋼・得到（或其他來源）
tags: [相關標籤]
date: YYYY-MM-DD（今天的日期）
---

# 模型名稱

## 一句話摘要
（從使用者的描述中提煉）

## 模型說明
（結構化整理使用者提供的內容）

## 使用場景
（從內容推導適用場景，或詢問使用者）

## 個人反思
（如果使用者有分享個人想法，記錄在此）

## 相關模型
（掃描現有模型後填寫，見步驟 4）
```

**補充現有模型：** 使用 Edit 工具更新對應區段。

### 4. 建立交叉連結

使用 Grep 搜尋所有 `models/` 檔案，找出與新模型概念相關的現有模型：
- 在新模型的「相關模型」區段加入連結
- 在相關的現有模型中也加入反向連結（使用 Edit）
- 連結格式：`- [模型名稱](相對路徑.md) — 一句話說明關聯`

### 5. 更新索引

讀取 `models/_index.md`，在對應分類下新增連結：
```markdown
- [模型名稱——一句話描述](分類/檔名.md)
```

更新最後更新日期。

### 6. 記錄到 log.md

在 `log.md` 末尾追加：

```markdown
## [YYYY-MM-DD] ingest | 模型名稱

- 來源：（萬維鋼・得到 / 文章 / 個人想法等）
- 分類：（所屬分類）
- 交叉連結：（列出新增的連結）
```

### 7. 清理 inbox（如適用）

如果素材來自 inbox/，整理完成後詢問使用者是否要刪除原始草稿。

## 報告結果

完成後向使用者報告：
- 建立/更新了哪個模型檔（含路徑）
- 新增了哪些交叉連結
- 索引是否已更新

## 注意事項

- 所有內容使用繁體中文（台灣）
- 保留使用者的原始用詞和觀點，不要過度改寫
- 如果使用者的描述不夠完整，寧可留空區段也不要捏造內容
- source 欄位要如實反映來源，不要全部預設為「萬維鋼・得到」

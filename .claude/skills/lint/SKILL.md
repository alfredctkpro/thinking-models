# Lint — 知識庫健檢

掃描整個知識庫，產出健檢報告，發現結構問題、缺漏和改善機會。

## 檢查項目

### 1. 孤立頁面（Orphan Pages）

掃描 `models/` 下所有 .md 檔案（排除 _template.md 和 _index.md），檢查每個模型是否被至少一個其他模型的「相關模型」區段連結。

**判定方式：** 用 Grep 搜尋該檔案的檔名是否出現在其他 .md 檔案中。

**輸出：** 列出沒有被任何模型連結的孤立頁面。

### 2. 空白區段（Empty Sections）

讀取每個模型檔，檢查以下區段是否有實質內容（非空白、非 HTML 註解）：
- 一句話摘要
- 模型說明
- 使用場景
- 個人反思
- 相關模型

**輸出：** 列出每個模型的空白區段。

### 3. 索引一致性（Index Consistency）

比對 `models/_index.md` 與實際存在的模型檔案：
- 索引中有但檔案不存在的連結（斷連）
- 檔案存在但索引中沒有的模型（遺漏）

**輸出：** 列出不一致的項目。

### 4. 交叉連結建議（Cross-link Suggestions）

讀取所有模型的標題、標籤和一句話摘要，分析哪些模型在概念上相關但尚未互相連結。

**判定方式：**
- 共享標籤但未互連
- 內容中提及另一個模型的關鍵概念但沒有連結
- 同一分類下的模型未互相參照

**輸出：** 建議新增的交叉連結，附簡短理由。

### 5. Frontmatter 完整性

檢查每個模型檔的 YAML frontmatter 是否包含必要欄位：
- title（必填）
- source（必填）
- tags（應有至少一個標籤）
- date（必填，格式 YYYY-MM-DD）

**輸出：** 列出 frontmatter 不完整的檔案。

## 輸出格式

產出結構化的健檢報告：

```markdown
# 知識庫健檢報告 — YYYY-MM-DD

## 📊 總覽
- 模型總數：X 個
- 日記總數：X 篇
- 健康分數：X/100

## 🔗 孤立頁面（X 個）
| 模型 | 路徑 |
|------|------|
| ... | ... |

## 📝 空白區段（X 處）
| 模型 | 空白區段 |
|------|----------|
| ... | 個人反思、使用場景 |

## 📑 索引不一致（X 處）
| 問題 | 詳情 |
|------|------|
| 遺漏 | ... |
| 斷連 | ... |

## 💡 交叉連結建議（X 條）
| 模型 A | 模型 B | 建議理由 |
|--------|--------|----------|
| ... | ... | ... |

## 📋 Frontmatter 問題（X 處）
| 檔案 | 缺漏欄位 |
|------|----------|
| ... | tags |

## 建議的優先修復
1. ...
2. ...
3. ...
```

## 參數說明

- `all`（預設）：執行所有檢查
- `links`：只檢查交叉連結相關（孤立頁面 + 交叉連結建議）
- `empty`：只檢查空白區段
- `orphans`：只檢查孤立頁面

## 注意事項

- 所有報告使用繁體中文（台灣）
- 報告僅提供建議，不自動修改任何檔案
- 如果使用者要求修復，再使用 Edit 工具逐一處理
- 健康分數計算：滿分 100，每個問題依嚴重程度扣分（孤立頁 -5、空白區段 -2、索引不一致 -10、frontmatter 缺漏 -3）
